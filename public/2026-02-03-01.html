基于 Hugging Face 免费服务器部署 OpenClaw 红龙虾教程

杆子 2026-02-03 09:31  教程 1628 字  4 分钟  编辑

【OpenClaw】 基于 Hugging Face 提供的免费 2CPU + 16GB 内存 + 50GB 硬盘服务器部署 OpenClaw 红龙虾

项目背景：开源"改名狂魔" OpenClaw

无论是 OpenClaw、Moltbot，还是最初的 Clawdbot，都是同一只可怜的红"龙虾"，堪称 2026 年开源圈最著名的"改名狂魔"兼"蜕壳专业户"。

改名三连冠

第一阶段：Clawdbot

项目刚诞生时叫 Clawdbot（爪爪机器人），名字巧妙融合了 Claw（龙虾爪）+ Claude（Anthropic 家的主力模型）的谐音梗，吉祥物是一只可爱的红龙虾。发布后瞬间火爆，GitHub Star 像坐火箭一样飙升。然而好景不长，Anthropic 的法务团队很快发来一封礼貌的"建议函"——言下之意：再不改名就法庭见。

第二阶段：Moltbot

吓得小龙虾当场第一次蜕壳，社区凌晨 5 点在 Discord 狂欢式脑暴，选中了 Moltbot 这个看似有深度的名字。Molt 意为龙虾/螃蟹蜕壳，寓意"抛弃旧壳才能长大"，既安全又有逼格。然而两天后发现，这名字念起来像在咳痰，连创始人都觉得拗口。

第三阶段：OpenClaw

不到三天，龙虾第二次（也是官方宣称的最后一次）华丽蜕壳，正式更名为 OpenClaw：

  • Open：开源、开放、社区驱动
  • Claw：保留龙虾血统，向最初的梗致敬

官方公告金句：“The lobster has molted into its final form."（龙虾已经蜕变为最终形态了）

翻译成大白话：“哥们儿终于长出硬壳了，这次不怕律师了！”

Hugging Face 简介

Hugging Face 提供了免费的云服务器资源：

  • 2 核 CPU
  • 16GB 内存
  • 50GB 硬盘

支持三种部署模板：Gradio、Docker、静态 HTML，可轻松部署各类应用。

部署步骤

第一步：登录 Hugging Face

  1. 访问 Hugging Face
  2. 点击 Log In 登录，已有账号直接输入用户名/邮箱和密码登录
  3. 没有账号点击 Sign Up 注册

第二步：创建 Space

  1. 点击 New > Space 创建空间

  1. 配置 Space 基本信息：
  • Space name：输入名称
  • Short description：可选填写描述
  • Docker：选择 Docker 模板（已默认选中 Blank）
  • Space hardware：已默认选中免费配置
  • Public：已默认选中公网访问

  1. 点击 Create Space 完成创建

第三步：修改 README.md 配置端口

  1. 进入 Space 工作台，点击 Files
  2. 点击 README.md 文件，再点击 Edit 编辑

  1. 在内容中添加 app_port: 18789：
---
app_port: 18789
---
  1. 向下滚动，点击 Commit changes to main 提交更改

第四步：创建 Dockerfile

  1. 点击 Files > Contribute > Create a new file

  1. 输入文件名：Dockerfile
  2. 内容输入：
FROM node:22-slim

  1. 点击 Commit new file to main 提交

第五步：等待自动构建部署

完成上述步骤后，Hugging Face Spaces 会自动开始构建和部署 OpenClaw。

部署状态说明：

状态含义处理方式
Building拉取镜像若出错，检查 Dockerfile 格式或镜像问题
Starting安装部署镜像若报错，查看 Container 日志排查
Running部署完成可正常访问使用

可通过点击 Log 查看构建日志跟踪进度。

第六步：配置环境变量

部署完成后查看日志，会发现因缺少环境变量而报错：

需要配置以下私密环境变量：

环境变量说明
OPENCLAW_GATEWAY_PASSWORD访问密码
GEMINI_APIKEYGoogle Gemini API Key

配置步骤：

  1. 进入 Space 工作台，点击 Settings
  2. 向下滚动找到 Variables and secrets
  3. 点击 New secret 添加私密环境变量

添加完环境变量后，系统会自动重新构建和部署 OpenClaw。

第七步：访问和使用

部署完成后，通过以下地址访问：

https://[YourName]-openclaw.hf.space/
  1. 点击设置旁边的按钮，选择 “Embed this Space” 获取访问链接

  1. 访问页面，正常显示 OpenClaw 界面

  1. 在 Gateway Token 中输入环境变量 OPENCLAW_GATEWAY_PASSWORD 的值，点击 Connect 查看连接状态

  1. 点击 App 即可开始使用
