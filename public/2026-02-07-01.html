背景介绍

在日常使用 Claude Code 时，当触发使用限额后，系统提示可以尝试 Extra Usage 模式，并附赠了 $50 的 API 额度。这种福利当然要及时领取！

如何查看

Extra Usage 额度 {#extrausage}

领取成功后，在界面中可以看到新增的 Extra Usage 区域。当你的小时限额或周限额触发后，系统会自动切换到该模式，按实际用量计费（Pay-as-you-go）。

使用成本实测

需要注意的是，这 $50 的额度消耗速度相当惊人。实测数据显示：在小时额度用尽的情况下，高强度编程约 1 小时，就消耗了 $20+ 的费用。由此看来，如果完全依赖按量付费模式，长期使用成本相当高昂，不适合作为常规开发方案。

使用建议

  1. 紧急情况使用：建议将 Extra Usage 作为限额触发后的应急方案，而非主力开发模式
  2. 成本监控：务必关注实时消费金额，避免额度快速耗尽
  3. 优化提示词：在高强度编码时，尽量精简上下文，减少 Token 消耗

总结：这 $50 的赠送额度非常适合在紧急赶工时作为”备用电池”，但日常开发还是建议合理分配免费额度，或等待限额重置。
