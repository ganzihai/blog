    <span leaf="">1. 简介</span>
  </h2>
  
  <p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
    <span leaf="">Gemini CLI 是 Google 发布的</span><strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">开源 AI Agent</span></strong><span leaf="">，将 Gemini 模型能力直接带入终端。它不是一个简单的聊天工具，而是一个具备 ReAct（推理-行动）循环的智能代理，能够：</span>
  </p>
  
  <ul class="list-paddingleft-1" style="list-style: circle;padding-left: 1em;margin-left: 0;color: #3f3f3f;">
    <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
      <section><span leaf="">• 读取、分析、修改本地文件和代码库</span></section>
    </li>
    <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
      <section><span leaf="">• 执行 Shell 命令（带权限控制）</span></section>
    </li>
    <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
      <section><span leaf="">• 调用 Google Search 进行联网搜索</span></section>
    </li>
    <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
      <section><span leaf="">• 抓取网页内容</span></section>
    </li>
    <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
      <section><span leaf="">• 通过 MCP 协议连接外部工具</span></section>
    </li>
    <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
      <section><span leaf="">• 理解图片、PDF、音频、视频等多模态输入</span></section>
    </li>
  </ul>

    <span leaf="">2.1 系统要求</span>
  </h3>
  
  <ul class="list-paddingleft-1" style="list-style: circle;padding-left: 1em;margin-left: 0;color: #3f3f3f;">
    <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
      <section><span leaf="">• </span><strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">Node.js</span></strong><span leaf="">：18.0 及以上版本（推荐 20+）</span></section>
    </li>
    <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
      <section><span leaf="">• </span><strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">操作系统</span></strong><span leaf="">：macOS、Linux、Windows（含 WSL）</span></section>

    <span leaf="">验证 Node.js 版本：</span>
  </p>
  
  <pre class="wp-block-code">`node --version  # 应输出 v18.x.x 或更高`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">2.2 安装方式</span>
</h3>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">方式一：npm 全局安装（推荐）</span></strong>
</p>

<pre class="wp-block-code">`npm install -g @google/gemini-cli`</pre>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">方式二：无需安装，直接使用 npx</span></strong>
</p>

<pre class="wp-block-code">`npx @google/gemini-cli`</pre>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">方式三：Conda 环境（数据科学场景）</span></strong>
</p>

<pre class="wp-block-code">`conda create -y -n gemini_env -c conda-forge nodejs
conda activate gemini_env
npm install -g @google/gemini-cli`</pre>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">方式四：安装预览版（最新功能，可能不稳定）</span></strong>
</p>

<pre class="wp-block-code">`npm install -g @google/gemini-cli@preview`</pre>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">方式五：每日构建版（最前沿，存在未验证问题）</span></strong>
</p>

<pre class="wp-block-code">`npm install -g @google/gemini-cli@nightly`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">2.3 升级</span>
</h3>

<pre class="wp-block-code">`npm update -g @google/gemini-cli`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">2.4 卸载</span>
</h3>

<pre class="wp-block-code">`npm uninstall -g @google/gemini-cli
# 同时清除配置（可选）
rm -rf ~/.gemini`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">2.5 验证安装</span>
</h3>

<pre class="wp-block-code">`gemini --version`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">2.6 配置目录结构</span>
</h3>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <span leaf="">Gemini CLI 的文件分布在以下位置：</span>
</p>

<pre class="wp-block-code">`~/.gemini/                         # 用户全局目录
├── settings.json                  # 用户级配置
├── GEMINI.md                      # 全局上下文文件
├── commands/                      # 用户自定义斜杠命令
│   └── my-command.toml
├── keybindings.json               # 自定义键位绑定
└── agents/                        # 本地 Agent 定义

&lt;project&gt;/
├── .gemini/                       # 项目级配置
│   ├── settings.json              # 项目设置（覆盖用户设置）
│   ├── commands/                  # 项目专属命令
│   │   └── review.toml
│   └── sandbox-macos-custom.sb   # 自定义沙箱配置
├── GEMINI.md                      # 项目上下文文件
└── .geminiignore                  # 忽略文件（类似 .gitignore）

# 系统级（企业管理员使用）
/etc/gemini-cli/settings.json      # Linux 系统级强制配置
C:\ProgramData\gemini-cli\settings.json  # Windows 系统级`</pre>

</h3>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <span leaf="">在提示词中使用 </span>`&lt;span leaf=""&gt;@路径&lt;/span&gt;`<span leaf=""> 将文件或目录内容注入上下文：</span>
</p>

<pre class="wp-block-code">`# 引用单个文件

# 引用 PDF
@spec.pdf 根据这个规格书生成实现代码

# 同时引用多个文件
@package.json @tsconfig.json 检查这两个配置是否有冲突`</pre>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">

</p>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">Git 感知</span></strong><span leaf="">：引用目录时，自动跳过 </span>`&lt;span leaf=""&gt;.gitignore&lt;/span&gt;`<span leaf=""> 和 </span>`&lt;span leaf=""&gt;.geminiignore&lt;/span&gt;`<span leaf=""> 中指定的内容。</span>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">

</p>

<pre class="wp-block-code">`# 执行并显示结果

!
# 现在在 Shell 模式中，直接输入命令
ls
git status

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">

<pre class="wp-block-code">`# 启动时开启
gemini --approval-mode plan

# 或在 settings.json 中设置
# "defaultApprovalMode": "plan"`</pre>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">使用场景</span></strong><span leaf="">：重构前先看执行计划，大型任务分析，不想意外修改文件。</span>
</p>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">11.2 Checkpointing（检查点）</span>
</h3>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">

<pre class="wp-block-code">`gemini --checkpointing "重构整个认证模块"

# 会话中查看检查点
/restore

# 回滚到指定检查点
/restore &lt;checkpoint_id&gt;`</pre>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">强烈推荐</span></strong><span leaf="">：进行重大代码变更前开启，等效于自动 git stash。</span>
</p>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">

# 管道输入
echo "解释这段代码" | gemini -p -

/settings  # 修改 defaultApprovalMode 为 auto_edit

# Step 2：实现
&gt; 按照上面的计划，先实现 TOTP 生成和验证的核心逻辑，
  放在 src/lib/totp.ts，并为其编写单元测试

# Step 3：集成
&gt; 将 TOTP 集成到 @src/auth/login.ts 的登录流程中

# Step 4：验证
&gt; 运行测试并确认所有用例通过`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">

  <span leaf="">15.1 安全注意事项</span>
</h3>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">认证安全</span></strong><span leaf="">：</span>
</p>

<ul class="list-paddingleft-1" style="list-style: circle;padding-left: 1em;margin-left: 0;color: #3f3f3f;">
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• 永远不要将 API Key 硬编码在代码或脚本中</span></section>
  </li>
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• 不要提交含有 API Key 的 </span>`&lt;span leaf=""&gt;.env&lt;/span&gt;`<span leaf=""> 文件</span></section>
  </li>
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• 在 </span>`&lt;span leaf=""&gt;.geminiignore&lt;/span&gt;`<span leaf=""> 中排除所有包含凭证的文件</span></section>
  </li>
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• 如果意外暴露了 API Key，立即在 Google AI Studio 中撤销并重新生成</span></section>
  </li>
</ul>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">环境变量陷阱</span></strong><span leaf="">：</span>
</p>

<ul class="list-paddingleft-1" style="list-style: circle;padding-left: 1em;margin-left: 0;color: #3f3f3f;">
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• </span>`&lt;span leaf=""&gt;GOOGLE_API_KEY&lt;/span&gt;`<span leaf="">、</span>`&lt;span leaf=""&gt;GEMINI_API_KEY&lt;/span&gt;`<span leaf="">、</span>`&lt;span leaf=""&gt;GOOGLE_GENAI_USE_VERTEXAI&lt;/span&gt;`<span leaf=""> 会覆盖 OAuth 登录</span></section>
  </li>
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• 想用免费 OAuth 时，确保这些变量没有被设置：</span></p> <pre class="wp-block-code">`unset GOOGLE_API_KEY GEMINI_API_KEY GOOGLE_GENAI_USE_VERTEXAI`</pre></section>
</li>
</ul>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">Yolo 模式风险</span></strong><span leaf="">：</span>
</p>

<ul class="list-paddingleft-1" style="list-style: circle;padding-left: 1em;margin-left: 0;color: #3f3f3f;">
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• </span>`&lt;span leaf=""&gt;--yolo&lt;/span&gt;`<span leaf=""> 会跳过所有确认提示，可能删除文件、执行危险命令</span></section>
  </li>
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• 生产机器上禁止使用 </span>`&lt;span leaf=""&gt;--yolo&lt;/span&gt;`</section>
  </li>
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• 重要任务配合 </span>`&lt;span leaf=""&gt;--checkpointing&lt;/span&gt;`<span leaf=""> 使用</span></section>
  </li>
</ul>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">15.2 文件操作注意事项</span>
</h3>

<ul class="list-paddingleft-1" style="list-style: circle;padding-left: 1em;margin-left: 0;color: #3f3f3f;">
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• </span><strong style="color: #0F4C81;font-weight: bold;font-size: inherit;"><span leaf="">始终开启检查点</span></strong><span leaf="">（</span>`&lt;span leaf=""&gt;--checkpointing&lt;/span&gt;`<span leaf="">）进行大范围文件修改</span></section>
  </li>
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• 修改前通过 </span>`&lt;span leaf=""&gt;--approval-mode plan&lt;/span&gt;`<span leaf=""> 先看执行计划</span></section>
  </li>
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• 用 </span>`&lt;span leaf=""&gt;.geminiignore&lt;/span&gt;`<span leaf=""> 排除不应被修改的目录（如 </span>`&lt;span leaf=""&gt;node_modules&lt;/span&gt;`<span leaf="">、</span>`&lt;span leaf=""&gt;dist&lt;/span&gt;`<span leaf="">、</span>`&lt;span leaf=""&gt;.git&lt;/span&gt;`<span leaf="">）</span></section>
  </li>
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• Gemini CLI 遵循 </span>`&lt;span leaf=""&gt;.gitignore&lt;/span&gt;`<span leaf=""> 规则，不会读取被忽略的文件</span></section>
  </li>
</ul>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">

<ul class="list-paddingleft-1" style="list-style: circle;padding-left: 1em;margin-left: 0;color: #3f3f3f;">

  </li>
  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">
    <section><span leaf="">• 长会话中使用 </span>`&lt;span leaf=""&gt;/compress&lt;/span&gt;`<span leaf=""> 压缩历史，节省 Token</span></section>

  <li style="display: block;margin: 0.2em 8px;color: #3f3f3f;">

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">

<pre class="wp-block-code">`# ❌ 模糊
&gt; 修复登录 bug

# ✅ 精确，带完成标准
&gt; 修复 src/auth/login.ts 中的登录 bug。
  完成标准：
  1. `pnpm test -- auth` 全部通过
  2. 没有新的 TypeScript 错误
  3. 不要修改 auth.test.ts 中的测试用例`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">技巧二：用 /init 快速生成项目上下文</span>
</h3>

<pre class="wp-block-code">`# 进入项目后第一件事
/init
# 然后根据实际情况编辑生成的 GEMINI.md`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">技巧三：@导入实现模块化 GEMINI.md</span>
</h3>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <span leaf="">主文件保持简洁，用 </span>`&lt;span leaf=""&gt;@&lt;/span&gt;`<span leaf=""> 导入各模块的详细规范：</span>
</p>

<pre class="wp-block-code">`# GEMINI.md 主文件
@./docs/coding-standards.md
@./docs/api-conventions.md
@./docs/testing-requirements.md`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">技巧四：Plan 模式先规划后确认</span>
</h3>

<p style="margin: 1.5em 8px;letter-spacing: 0.1em;color: #3f3f3f;">
  <span leaf="">对于复杂任务，先用 Plan 模式确认方向：</span>
</p>

<pre class="wp-block-code">`gemini --approval-mode plan
&gt; 重构整个支付模块

# 看到计划后确认无误，再切换到执行模式
/settings  # 改为 auto_edit`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">技巧五：善用 /compress 控制 Token</span>
</h3>

<pre class="wp-block-code">`# 长会话中，定期压缩
/compress
# Gemini 用摘要替换完整历史，继续任务`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">技巧六：Checkpointing + Rewind 的黄金组合</span>
</h3>

<pre class="wp-block-code">`gemini --checkpointing

# 出现问题时
/rewind          # 回退到上一步
# 或
/restore         # 查看所有检查点，选择回退点`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">

</h3>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">

  分析他们的行为模式，生成一份报告`</pre>

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">
  <span leaf="">高频斜杠命令</span>
</h3>

<pre class="wp-block-code">`/clear          清空对话和屏幕

/model          切换模型
/memory show    查看当前上下文
/memory add     添加记忆
/restore        回滚到检查点
/rewind         撤销上一次工具操作

<h3 data-heading="true" style="padding-left: 8px;border-left: 3px solid #0F4C81;margin: 2em 8px 0.75em 0;color: #3f3f3f;font-size: 15.4px;font-weight: bold;line-height: 1.2;">

<pre class="wp-block-code">`@文件路径        引用文件内容
