#### 前言 {#}

大家好，今天分享一个使用 HuggingFace 免费搭建 **100G 图床和网盘** 的教程，**无需实名认证**即可完成搭建。

之前写过关于使用缤纷云搭建图床的教程，很多小伙伴反馈需要实名认证，担心隐私信息安全。因此特意寻找了替代方案。今天分享的项目是 **CloudFlare-ImgBed**，这是一个基于 Cloudflare Pages 的开源文件托管解决方案，致力于为用户提供**免费、稳定、高效**的文件存储服务。

该项目支持多种存储渠道，兼容无服务器和有服务器多种部署方式。目前该项目在 GitHub 上已获得 4k+ Star，由 **MarSeventh** 大佬开发，支持多种存储方式且配置灵活。

#### 技术架构 {#-2}

该系统采用前后端分离设计：

  * **前端**：基于 Vue.js 实现响应式界面。
  * **后端**：依托 Cloudflare Workers 搭建无服务器 API。
  * **存储层**：兼容 Telegram、R2、S3、Discard、HuggingFace 等多类后端。
  * **元数据**：统一通过 Cloudflare KV 管理。

#### 项目部署 {#-3}

作者提供了两种部署方式：**Cloudflare Pages 托管**（推荐）和 **Docker 部署**。

首先，需要 Fork 源码仓库到自己的 GitHub，以便后续更新可直接同步部署。

> **GitHub 地址**：https://github.com/MarSeventh/CloudFlare-ImgBed

##### Cloudflare Pages 部署（推荐） {#cloudflarepages}

**1. 创建项目**  
在控制面板找到【计算和 AI】，点击【Workers and Pages】，在页面右上角点击【创建应用程序】。点击“想要部署 Pages？”下的【开始按钮】。

![image-20260116123017642](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJmhNf4yiaYFdib3Z8BBjlYUCdr5Uo7HPciaPcvmyMFaSvEYccicQcwf6YX3A/640?wx_fmt=png&#038;from=appmsg) 

**2. 导入代码**  
在“导入现有 Git 存储库”处点击“开始使用”。

![image-20260116123114028](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJm6L45yc47uq3I6XRFvAB3Z2wakzxZ5FGWnhN9mCkAENlMLpfpib2Lr8A/640?wx_fmt=png&#038;from=appmsg) 

**3. 选择仓库**  
选择你 Fork 的 **CloudFlare-ImgBed** 项目，点击【开始设置】。

![image-20260116123254097](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJmRx7Mslrp7SccTIfbDTCa7Y8GVRy8iazc9rxVLlwFxImOicAugic3X2DDA/640?wx_fmt=png&#038;from=appmsg) 

**4. 构建配置**  
项目名称自定义，**构建命令**填入：`npm install`，其他保持默认，点击【保存并部署】。

![image-20260116123625570](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJmYuI3fFus2YykXYzXaypLqDtMXr9ouWvZE8ia3aJtGPwdDXSm093da5A/640?wx_fmt=png&#038;from=appmsg) 

**5. 配置数据库**

> **注意**：KV 数据库和 D1 数据库**只需配置其中一个**即可，建议根据需求选择。

  * **KV 数据库**：进入【存储和数据库】 -> 【Workers KV】，添加 KV 数据库，KV 名称填写 **img_url**。

![image-20260116124052752](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJm4NWWpMKMFb5dqT4tJu1KRbVaZ8ynbVv4lOvRDMu57DwSbhmydSWBBA/640?wx_fmt=png&#038;from=appmsg) 

  * **D1 数据库**：进入【存储和数据库】 -> 【D1 SQL 数据库】，点击右上角【创建数据库实例】。

**6. 绑定数据库**  
在 Workers and Pages 中点击刚刚部署好的项目，进入【设置】 -> 【绑定】。添加 KV 数据库绑定如下：

  * **变量名称**：`img_url`
  * **KV 命名空间**：`img_url`

![image-20260116124628645](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJmBrrEuecbGicLFexgMjiaicZJldaCjkWVMbCWxMwDHJpJoVcmeYnTTHCgA/640?wx_fmt=png&#038;from=appmsg) 

点击保存后，项目会自动重新部署。完成后即可使用 Cloudflare 分配的域名访问，建议绑定自己的自定义域名。

##### Docker 方式部署 {#docker}

**1. 创建项目目录**

```
mkdir -p /workspace/cloudflare-imgbed
cd /workspace/cloudflare-imgbed

```

**2. 下载 docker-compose.yml 文件**

```
wget https://raw.githubusercontent.com/MarSeventh/CloudFlare-ImgBed/main/docker-compose.yml

```

**3. 创建配置文件**

```
sudo tee /workspace/cloudflare-imgbed/wrangler.toml &lt;&lt;-&#039;EOF&#039;
name = "cloudflare-imgbed"
compatibility_date = "2024-07-24"
EOF

```

**4. 启动服务**

```
docker compose up -d

```

服务启动后，访问 **7658** 端口即可。如果是服务器部署，请务必开放防火墙或配置端口转发。

#### HuggingFace 注册与配置 {#huggingface}

> **官方地址**：https://huggingface.co/

图片存储将使用 HuggingFace 的 **Repositories Storage**。HuggingFace 渠道支持大文件直传（>20MB），系统会自动使用 LFS 协议进行分片上传。

**1. 创建新空间**  
注册并登录后，在控制面板新建一个 New Space。

![image-20260116132250472](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJmFVicibicB12JHEO1jHFcZEqia9INZYaibaUttRCnABicTyHWzicmDficp0xITg/640?wx_fmt=png&#038;from=appmsg) 

**2. 配置空间**

  * **Space name**：自定义。
  * **Select the Space SDK**：选择 **Docker** 的空项目或其他皆可。
  * **Space hardware**：选择 **Free**（免费版）。

![image-20260116132506144](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJm1DlahdvzrgmOZY4bNVib4P6iaicP9wruWqfAYCqBbjjBgs0a9WOBMyFFw/640?wx_fmt=png&#038;from=appmsg) 

> 空间 SDK 选择任意类型均可，因为我们需要的是一个仓库标识。空间可见性可根据个人意愿选择 Public 或 Private。

点击【Create Space】创建。

![image-20260116132730798](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJmhco1ZV0F0zz4wxNztBYaSPDQRkWhgGh9A5ZKaAs8cMhOEeULmtbrWw/640?wx_fmt=png&#038;from=appmsg) 

**3. 创建 Access Token**  
进入 【Settings】 -> 【Access Tokens】，点击【Create new token】。

  * **Token name**：自定义。
  * **权限**：务必勾选 **Role** 为 **Write**（仓库读写权限）。

> **链接直达**：https://huggingface.co/settings/tokens

![image-20260116133152748](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJm8gVAC5iagiajFQTTdL8SlWicFuOrNXQiayiaKf5wJ6iaia2WGHh61fCnFLKug/640?wx_fmt=png&#038;from=appmsg)

#### 存储配置 {#-4}

**1. 添加上传渠道**  
进入 CloudFlare-ImgBed 系统后台，点击【系统设置】 -> 【上传设置】，在添加上传渠道区域进行配置。

![image-20260116133429256](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJmMmqiaw9xXiafGAGuQnDTgDnBxPVDibNUAnGUHycX51u9LCToDmibibMMrbQ/640?wx_fmt=png&#038;from=appmsg) 

**2. 配置 HF 存储参数**

  * **渠道名称**：自定义。
  * **仓库名称**：必须严格遵循 **`用户名/仓库名`** 的格式（即 HuggingFace 上的 Space 路径）。
  * **Access Token**：填入上一步申请的 Token。

点击保存。

![image-20260116133549982](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJmzW8s3Z24PoGq8NMica2AeckI8U65ZUgoXKvLBQSLPxrBcJJsuziaBulg/640?wx_fmt=png&#038;from=appmsg) 

**3. 设置默认上传渠道**  
在【系统设置】 -> 【页面设置】中，配置默认上传渠道及其他个性化设置。

![image-20260116133833507](https://wsrv.nl/?output=webp&#038;url=https://mmbiz.qpic.cn/mmbiz_png/lUaUbFZtsVJ5z5hhIPL9K7uVvhs37sJmD50rVvUdB8wh882uiaQ7NOxv3wFFeog0olB51UUX7SO2ibUV3ibpGykAw/640?wx_fmt=png&#038;from=appmsg) 

完成以上配置，即可开始使用你的专属图床和网盘。
