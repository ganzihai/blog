  <p data-path-to-node="4">
    对于很多使用 Adobe、Autodesk 等大型专业软件的创作者或开发者来说，C 盘空间总是不知不觉被吞噬。直接复制粘贴转移往往会因为“权限丢失”或“系统服务占用”导致软件彻底罢工。
  </p>
  
  <p data-path-to-node="5">
    本文记录了一套极其硬核且万无一失的 C 盘深度清理方案，从底层的 `Robocopy` 克隆、NTFS 软链接，到系统环境隔离，帮你把 C 盘打造成一个纯粹的系统启动盘。
  </p>
  
  <h2 data-path-to-node="6">
    一、 核心原理与核心工具
  </h2>
  
  <p data-path-to-node="7">
    我们的最终目标是将 C 盘里那些庞大且不可更改路径的隐藏文件夹（如 `ProgramData` 下的软件数据）转移到 D 盘。但为了“骗”过软件，让它们依然以为数据在 C 盘，我们需要用到 <b data-path-to-node="7" data-index-in-node="93">NTFS 目录联接（Junction）</b>。
  </p>
  
  <ul data-path-to-node="8">
    <li>
      <p data-path-to-node="8,0,0">
        <b data-path-to-node="8,0,0" data-index-in-node="0">克隆工具：`robocopy`</b>。相比于普通的复制粘贴，它能带上特定的参数，完美克隆原文件夹的所有读写权限（ACLs）、所有者和时间戳，确保转移后软件不会报权限错误。
      </p>
    </li>
    
    <li>
      <p data-path-to-node="8,1,0">
        <b data-path-to-node="8,1,0" data-index-in-node="0">链接工具：`mklink /J`</b>。创建一个底层目录联接，相当于给 C 盘开了一个“任意门”，数据实际存在 D 盘，但软件只管往 C 盘写，系统底层会自动重定向。
      </p>
    </li>
  </ul>

  <h3 data-path-to-node="12">
    为什么必须进“安全模式”？
  </h3>
  
  <p data-path-to-node="13">
    正常系统下，Adobe 和 Autodesk 的后台流氓服务会死死占用这些文件，且重启后会自动生成新文件夹，导致软链接创建失败。在安全模式下，所有第三方服务被强制休眠，是我们动手的绝佳时机。
  </p>

<h2 data-path-to-node="17">

<h3 data-path-to-node="19">
  1. 系统/软件临时垃圾（Temp）：修改环境变量
</h3>

<p data-path-to-node="20">
  不需要敲代码，用微软官方途径更稳妥：
</p>

<ol start="1" data-path-to-node="21">
  <li>
    <p data-path-to-node="21,0,0">
      在 D 盘新建 `D:\Temp_Cache`。
    </p>
  </li>
  
  <li>
    <p data-path-to-node="21,1,0">
      搜索并打开“编辑系统环境变量” -> 环境变量。
    </p>

  <li>
    <p data-path-to-node="21,2,0">
      将“用户变量”和“系统变量”里的 `TEMP` 和 `TMP`，全部修改为 `D:\Temp_Cache`。
    </p>

  <li>
    <p data-path-to-node="21,3,0">
      <b data-path-to-node="21,3,0" data-index-in-node="0">防背刺技巧：</b> 重启电脑后，去原路径 `C:\Users\Gan\AppData\Local\Temp` 和 `C:\Windows\Temp` 清空里面的所有文件，但<b data-path-to-node="21,3,0" data-index-in-node="79">千万保留这两个空文件夹本身</b>，以防那些写死了绝对路径的“老古董”软件找不到家而崩溃。

<h2 data-path-to-node="25">
  四、 绝对禁区：Fonts 字体文件夹的安全玩法
</h2>

<p data-path-to-node="26">
  作为设计师，安装几百上千款字体很正常，但 <b data-path-to-node="26" data-index-in-node="21">绝对不要试图将 `C:\Windows\Fonts` 移动或做软链接！</b> Windows 图形界面（包括开机画面）在极早期强依赖该路径，一旦软链接解析延迟，系统直接显示乱码豆腐块甚至蓝屏。
</p>

<p data-path-to-node="27">
  <b data-path-to-node="27" data-index-in-node="0">优雅管理字体的方案：</b>
</p>

<ol start="1" data-path-to-node="28">
  <li>
    <p data-path-to-node="28,0,0">
      <b data-path-to-node="28,0,0" data-index-in-node="0">原生快捷方式安装：</b> 在控制面板的“字体设置”中勾选“允许使用快捷方式安装字体”。然后把你的 `.ttf/.otf` 源文件放在 D 盘，全选后右键 -> <b data-path-to-node="28,0,0" data-index-in-node="75">作为快捷方式安装</b>。
    </p>
  </li>
  
  <li>
    <p data-path-to-node="28,1,0">
      <b data-path-to-node="28,1,0" data-index-in-node="0">如何剥离已安装的几百个字体？</b> `C:\Windows\Fonts` 是个伪装的外壳，普通界面看不出时间。用解压软件（如 360zip/WinRAR）在地址栏输入 `C:\Windows\Fonts`，就能看到真正的文件列表。按“修改日期”倒序排列，将你近期安装的字体“剪切”到 D 盘即可。（注意：`Segoe UI`、`微软雅黑`、`SimSun` 等系统保命字体千万别碰！）
    </p>
  </li>
</ol>

  <li>
