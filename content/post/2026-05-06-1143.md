**结构化 Prompt 模板：四个维度榨干模型能力**

经验表明，混沌的自然语言往往让模型生成结果偏差很大。一套成熟的 Prompt 应包含四个要素：主体、环境、风格、用途。可复用的模板如下：

```text
[主体] + [环境/背景] + [风格/配色] + [应用场景提示]
```

举一个开发场景的真实例子：为微服务架构文档制作配图。

*   **混沌版 Prompt**：“画一张微服务架构图。” → 输出可能是一堆模糊的连线，甚至直接给出网络示意图。
*   **结构化 Prompt**：
    *   主体：多个微服务模块通过API网关连接，服务之间用箭头表示调用关系。
    *   环境背景：放在纯白背景上，整体布局横向。
    *   风格：扁平化、企业IT风格，主色为科技蓝+灰色。
    *   用途：希望作为技术文档插图，避免文字过多，留白适合后期注释。

实测中将该 Prompt 输入 kulaai 的 GPT-5.5 对话窗口，3.2 秒返回的图片已经接近一页 PPT 底稿，只需轻度标注即可使用。相同 Prompt 在 Gemini 3 Pro 上用时 2.9 秒，结构更整洁，但艺术感稍弱。这样一对比就能根据场景快速选模型。

**实战：用 kulaai 完成技术博客全套配图**

假设你要写一篇《Rust 异步运行时深度解析》，需要封面图、架构示意图、性能对比图共三张图片，下面展示用 kulaai 一次性解决的完整流程。

**1. 封面图生成**
使用 GPT-5.5 模型，输入：
> 生成一张宽屏技术博客头图，主体为锈色金属齿轮与异步任务流箭头，背景深蓝色科技网格，右下角留空白用于放置标题，风格硬朗，适合 CSDN 头图。

首张图 3.1 秒返回，齿轮和箭头形态准确，留白区域也很合适。

**2. 架构示意图**
切换到 Gemini 3 Pro，输入：
> 一张横向架构图：三个方框分别代表“任务队列”“工作线程池”“I/O 多路复用器”，用方向箭头连接，图标风格，白底蓝框，去掉文字。

Gemini 的 2.8 秒输出干净整齐，可以直接作为文档插图，避免了从 Visio 开始画的麻烦。

**3. 性能对比图**
如果需要数据图表，可以先上传一份 CSV 文件（包含不同异步框架的测试数据），然后对 GPT-5.5 说：
> 根据这份 CSV 数据，生成一张柱状图，比较三种框架的吞吐量，要求图表颜色为蓝、橙、绿，风格学术化，X 轴标签字号适中。

模型会先读取文件，再输出图像。实测此过程约 4.5 秒，生成的图表与 Excel 自动绘图效果接近，足以满足内部分享。

**避坑指南与效率技巧**

1.  **禁止出现真实文字**：目前模型对中英文字符的渲染仍有缺陷，图表内如果需要标签，可以先让模型生成图像，再用其他工具添加，或者明确要求“不要放置任何文字”。
2.  **用“用途”控制留白**：Prompt 里加上“用作文章头图，预留三分之一右侧空白”或“左对齐构图”，可以大幅度减少后期裁切工作量。
3.  **同一对话持续修改**：kulaai 保留对话上下文，第一次出图后，直接回复“把背景调亮”“增加两个人像”即可，无需重新输入整个 Prompt。
4.  **结合联网搜索获取实时参考**：需要生成模拟真实数据的图表时，可以先用 kulaai 的联网搜索功能获取最新统计信息，再在同一对话内要求模型依据这些信息出图。
5.  **监控每日免费额度**：在平台界面上通常有额度提示。轻度博客更新（每天 3~5 张图）完全在免费范围内。
