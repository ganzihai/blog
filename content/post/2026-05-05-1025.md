* **三款可用模型**，每 5 小时 1500 次调用
* 每个模型独立计数，同账户所有 Key 共享配额
* 最多可创建 20 个 API Key

### 三款模型一览

| 模型 | 定位 |
| :--- | :--- |
| **SenseNova 6.7 Flash-Lite** | 轻量多模态智能体模型，面向真实工作流 |
| **SenseNova U1 Fast** | 多模态理解与生成一体模型，专供信息图生成 |
| **DeepSeek V4 Flash** | 高性能对话模型，支持思考/非思考模式、256K 上下文 |

三款模型各有分工：Flash-Lite 适合日常办公流，U1 Fast 适合多模态内容创作，DeepSeek V4 Flash 则适合需要长上下文理解的高级任务。

#### SenseNova 6.7 Flash-Lite

面向真实工作流的轻量多模态智能体模型，支持文本对话与图像输入理解。

* 轻量高效，兼顾效果、成本与落地性
* 原生多模态架构，支持图像输入理解（OCR、图表解读等）
* 办公场景增强，稳定支撑复杂长链路任务
* Token 效率更优，复杂任务成本更可控
* 上下文长度 256K tokens（最大输入 252K，最大输出 64K）
* **MODEL ID**: `sensenova-6.7-flash-lite`

#### SenseNova U1 Fast

SenseNova U1 Fast 基于 SenseNova U1 的加速版本，专供信息图（Infographics）生成场景。

* **MODEL ID**: `sensenova-u1-fast`
* **注意**：U1 Fast 使用独立的图像生成接口 `POST /v1/images/generations`，不是 Chat Completions；不支持图像输入。

#### DeepSeek V4 Flash

DeepSeek 高性能对话模型，支持思考模式与非思考模式，上下文长度 256K tokens，最大输出 64K tokens，内置 JSON Output、Tool Calls 等功能。

* **MODEL ID**: `deepseek-v4-flash`

### 调用方式 Python 示例

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://token.sensenova.cn/v1",
    api_key="{your key}",
)

resp = client.chat.completions.create(
    model="sensenova-6.7-flash-lite",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(resp.choices[0].message.content)
```

## 怎么接入？

Base URL：`https://api.sensenova.cn/v1`

兼容 OpenAI 格式，现有 OpenAI SDK 或 Agent 工具（如 OpenClaw）可以直接迁移，Key 数量上限 20 个。对于已经在用 OpenClaw 的用户来说，配一个商汤的 Key 作为补充调用渠道，成本几乎为零。
