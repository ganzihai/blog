在 Oracle Corporation 的云服务（OCI）中，一台服务器最多可以绑定多个公网 IP（通常上限约 10 个）。这个功能非常适合：多账号运营、推流、代理、分业务部署等场景。

## 一、效果说明 {#}

完成后你将拥有：

  * **1 台服务器**
  * **1 个主公网 IP + 9 个附加公网 IP（共最多 10 个）**
  * 每个 IP 都可以单独使用（独立访问 / 推流 / 请求）

* * *

## 二、核心原理（必须懂） {#-1}

OCI 网络结构：

```
VNIC（网卡）
├── Primary Private IP（主私网IP） → 绑定公网IP
└── Secondary Private IP（多个） → 每个可绑定公网IP

```

> 💡 **关键点：** 一个公网 IP ≠ 直接加，必须先创建私网 IP（Private IP），再绑定公网 IP

* * *

## 三、操作步骤 {#-2}

### 1️⃣ 进入实例 {#1}

  1. 登录 OCI 控制台
  2. 进入：Compute → Instances → 你的实例

### 2️⃣ 打开网卡（VNIC） {#2vnic}

  1. 点击：Attached VNICs
  2. 点进去主网卡（通常是 eth0）

### 3️⃣ 添加私网 IP {#3ip}

  1. 点击：IP Addresses → Assign Private IP Address
  2. 重复操作，添加多个

  * 建议添加 9 个（加上主 IP = 10）

### 4️⃣ 绑定公网 IP {#4ip}

对每一个 Private IP：

  1. 点击该 IP
  2. 选择：Edit → Assign Public IP
  3. 选择相应选项

### 5️⃣ 完成 {#5}

现在你会看到：多个 Public IP 已经挂在同一台机器上 ✅

* * *

## 四、服务器配置（重点！） {#-3}

默认系统不会自动使用这些 IP，需要手动配置。

### Linux（Ubuntu / Debian） {#linuxubuntudebian}

**查看 IP**

```
ip a

```

**添加 IP（临时）**

```
sudo ip addr add 公网IP/24 dev eth0

```

例如：

```
sudo ip addr add 129.xxx.xxx.xxx/24 dev eth0

```

**永久生效（推荐）**

编辑：

```
sudo nano /etc/netplan/00-installer-config.yaml

```

添加：

```
addresses:
  - 10.0.0.2/24
  - 10.0.0.3/24
  - 10.0.0.4/24

```

然后：

```
sudo netplan apply

```

* * *

## 五、验证是否成功 {#-4}

用不同 IP 请求：

```
curl ifconfig.me

```

或指定出口 IP：

```
curl --interface 你的IP ifconfig.me

```

## 六、实际应用场景 {#-5}

### 🎥 1. 多路推流（YouTube / OBS） {#1youtubeobs}

  * 每个 IP 推一个流
  * 避免限速/限流

### 🌐 2. 多账号运营 {#2}

  * 每个账号固定一个 IP
  * 降低关联风险

### 🔁 3. 代理 / 中转节点 {#3}

  * HTTP / SOCKS5 / Shadowsocks

### 🚀 4. 负载分流 {#4}

  * 不同业务走不同 IP

* * *

## 七、注意事项 {#-6}

> ⚠️ **1. 带宽是共享的**
> 
>   * 不是 10 倍带宽！
> 
> ⚠️ **2. IP 不一定全能申请到**
> 
>   * 有区域限制（东京 / 新加坡紧张）
> 
> ⚠️ **3. 免费账号限制**
> 
>   * Always Free 可能：
>   * IP 数量有限
>   * 容易被回收
> 
> ⚠️ **4. 端口默认是通的，但建议检查**
> 
>   * 安全列表（Security List）
>   * 防火墙（iptables / ufw）

* * *

## 八、进阶玩法 {#-7}

如果你想更专业：

  * 每个 IP 绑定不同端口
  * 每个 IP 跑独立 Docker 容器
  * 配合：
  * Nginx 多出口
  * iptables 分流
  * ffmpeg 多推流

* * *

## 九、总结一句话 {#-8}

> 👉 甲骨文云的&#8221;10 IP&#8221;本质是：**多个私网 IP + 每个绑定一个公网 IP**，实现一机多出口
