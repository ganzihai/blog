> 
> 但是多了一个使用 **Cloudflare** 的理由

![cover_image](https://wsrv.nl/?output=webp&url=https://mmbiz.qpic.cn/mmbiz_jpg/Ts4Edaib9xxeoibrAqicbjFG6BTY6KVY9bYaia3cU1DiapX4iaIkbYC8e7CHUic11ddQKE75UOpCy5TmZZ3RibQicVZjEdTEPkEWGiapSicXYMGQ7CL2FE/0?wx_fmt=jpeg) 

Bitwarden 是少数客户端与服务器端都开源的密码管理系统，支持完整自托管部署。[@Appinn][1]

但有人更进一步：直接把服务器端运行在 **Cloudflare Workers** 上——也就是说，你连 VPS 都可以省了。

![NodeWarden - 把 Bitwarden 搬上 Cloudflare Workers，彻底告别服务器 1](https://wsrv.nl/?output=webp&url=https://mmbiz.qpic.cn/mmbiz_jpg/Ts4Edaib9xxfBibZBU1ic142CSVX0rtia2S0K2Y5IQuxbCRB8rib4j13RjcswUAW57g52L1XS0N55xicfeich2Ot1rAvkZumqpLCScssmBhAkGklaw/640?wx_fmt=jpeg&from=appmsg) 

部署 NodeWarden 之后的效果，就是在无服务器的情况下，也能在手机、电脑上使用 Bitwarden 客户端来保存密码了，支持自动登陆、二次验证之类的功能。

## NodeWarden 与 Bitwarden 区别 {#nodewardenbitwarden}

| 能力项 | Bitwarden | NodeWarden | 说明 |  
|&#8212;&#8212;&#8211;|&#8212;&#8212;&#8212;&#8211;|&#8212;&#8212;&#8212;&#8212;|&#8212;&#8212;|  
| 单用户保管库（登录/笔记/卡片/身份） | ✅ | ✅ | 基于 Cloudflare D1 |  
| 文件夹 / 收藏 | ✅ | ✅ | 常用管理能力可用 |  
| 全量同步 `/api/sync` | ✅ | ✅ | 已做兼容与性能优化 |  
| 附件上传/下载 | ✅ | ✅ | 基于 Cloudflare R2 |  
| 导入功能 | ✅ | ✅ | 覆盖常见导入路径 |  
| 网站图标代理 | ✅ | ✅ | 通过 `/icons/{hostname}/icon.png` |  
| Passkey、TOTP | ❌ | ✅ | 官方需要会员，我们的不需要 |  
| 多用户 | ✅ | ❌ | NodeWarden 定位单用户 |  
| 组织/集合/成员权限 | ✅ | ❌ | 没必要实现 |  
| 登录 2FA（TOTP/WebAuthn/Duo/Email） | ✅ | ⚠️ 部分支持 | 仅支持 TOTP（通过 `TOTP_SECRET`） |  
| SSO / SCIM / 企业目录 | ✅ | ❌ | 没必要实现 |  
| Send | ✅ | ❌ | 基本没人用 |  
| 紧急访问 | ✅ | ❌ | 没必要实现 |  
| 管理后台 / 计费订阅 | ✅ | ❌ | 纯免费 |  
| 推送通知完整链路 | ✅ | ❌ | 没必要实现 |

## 必要条件 {#-1}

  1. 你需要有一个 **Cloudflare 账号**（必须有一个域名和信用卡）
  2. 一个 **GitHub 账号**

### 设置 NodeWarden {#nodewarden}

部署成功之后，Cloudflare 会提供一个临时地址，类似 `1nodewarden.apipnn.workers.dev`，用浏览器打开它。如果打不开，可以绑定一个你自己的二级域名。

![NodeWarden - 把 Bitwarden 搬上 Cloudflare Workers，彻底告别服务器 4](https://wsrv.nl/?output=webp&url=https://mmbiz.qpic.cn/sz_mmbiz/Ts4Edaib9xxeFOERj0lmHRKmib3TyrvEdBjD4PjficnSEKtFveRg1tNeRIfibTRRxMedxMhQ8LC1Yf7pSlsYjazgMuZXY8U5rAjPVpKTQFkCUUE/640?wx_fmt=other&from=appmsg) 

根据页面提示，一步一步进行即可。这个步骤主要包括：

  1. 设置 **JWT_SECRET**
  2. 设置自动更新 GitHub
  3. 设置主账号与密码
  4. 设置启用主账号的二次验证

![NodeWarden - 把 Bitwarden 搬上 Cloudflare Workers，彻底告别服务器 5](https://wsrv.nl/?output=webp&url=https://mmbiz.qpic.cn/mmbiz/Ts4Edaib9xxdbJmGTWF6tHPdWicRoZkZZzQzL8MxJbX49tSrfpnakZbKkXrcyPrV95CxPakic7xNxbkGibutYuks1VUjz0S58t4sBAyf2CLbrbg/640?wx_fmt=other&from=appmsg) 

最后一步成功之后，还能选择彻底隐藏这个设置页面：

![NodeWarden - 把 Bitwarden 搬上 Cloudflare Workers，彻底告别服务器 6](https://wsrv.nl/?output=webp&url=https://mmbiz.qpic.cn/mmbiz/Ts4Edaib9xxcX0DIbbSpJDVPJtezU0CMXQ27D8LTwXCqMKUEXibFx2m1nkYDndK8PRPvic0Rp0GoV3uib33X8qzPIbDgYwLvLEKBK1cP74bZH9k/640?wx_fmt=other&from=appmsg)

## 关于密码安全 {#-5}

**Bitwarden 的设计是这样的**：密码在本地加密，服务器只存储加密后的密文，主密码不上传服务器（服务器只保存验证值）。

所以，理论上服务器无法解密你的数据。而部署在 Cloudflare 上是相同的，只要最开始的代码没有恶意，就没有问题。因此**开源很重要**，如果有恶意代码，会很快被人发现。
